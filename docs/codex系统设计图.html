系统架构

前后端分离：React 19 + TypeScript + Vite 前端，Spring Boot 3.3 + MyBatis + Flyway + Spring Security 后端。自定义 Bearer Token（7 天），PasswordEncoder 加密存储；MySQL campus_share，服务端口 8085。

API 调用流

前端 request.ts 注入 Bearer；后端 TokenAuthenticationFilter 查 tokens 表；401 清 token 并跳登录。

sequenceDiagram
  participant U as 用户
  participant P as Login 页面
  participant R as request.ts
  participant API as AuthController
  participant S as AuthService
  participant DB as MySQL
  U->>P: 输入用户名/密码
  P->>R: POST /login
  R->>API: JSON 登录请求
  API->>S: login()
  S->>DB: 查 User + 校验密码
  DB-->>S: User
  S->>DB: 创建 Token(7天)
  S-->>API: token + user
  API-->>R: ApiResponse
  R-->>P: 登录成功
  P->>P: localStorage 保存 token
        
sequenceDiagram
  participant Page as 业务页面
  participant R as request.ts
  participant F as TokenFilter
  participant C as RestController
  participant S as Service
  participant DB as MySQL
  Page->>R: get/post/put/patch
  R->>F: /api/* + Bearer token
  F->>DB: 查 Token / 过期 / User
  F-->>C: SecurityContext
  C->>S: 业务方法
  S->>DB: MyBatis Mapper
  DB-->>S: 实体/聚合
  S-->>C: DTO / Map
  C-->>R: ApiResponse / PageResult
  R-->>Page: 渲染

数据模型

表结构由 Flyway V1__init.sql 管理；商品与图片支持 is_deleted 软删除。

erDiagram
  USER ||--o{ TOKEN : owns
  USER ||--o{ PRODUCT : sells
  CATEGORY ||--o{ PRODUCT : classifies
  PRODUCT ||--o{ PRODUCT_IMAGE : has
  USER ||--o{ APPOINTMENT : buyer
  PRODUCT ||--o{ APPOINTMENT : reserved
  USER ||--o{ FAVORITE : collects
  PRODUCT ||--o{ FAVORITE : collected
  USER ||--o{ COMMENT : writes
  PRODUCT ||--o{ COMMENT : receives
  USER { bigint id PK string username UK string password boolean is_active boolean is_staff datetime created_at }
  TOKEN { string key PK bigint user_id FK datetime created_at datetime expires_at }
  CATEGORY { bigint id PK string name UK int sort_order datetime created_at }
  PRODUCT { bigint id PK string title text description decimal price bigint category_id FK bigint seller_id FK string status boolean is_deleted }
  PRODUCT_IMAGE { bigint id PK bigint product_id FK string image boolean is_deleted }
  APPOINTMENT { bigint id PK bigint buyer_id FK bigint product_id FK string status }
  FAVORITE { bigint id PK bigint user_id FK bigint product_id FK }
  COMMENT { bigint id PK bigint user_id FK bigint product_id FK string content }
